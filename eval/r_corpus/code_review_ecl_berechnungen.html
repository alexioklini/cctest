Zweck der Skripte

Die Skripte enthalten Funktionen zur Berechnung des Expected Credit Loss (ECL)

für Finanzportfolios. Dabei werden verschiedene Methoden wie

- 12-Monats-ECL (get_12M_ECL),

- Langfristiger ECL (get_LT_ECL, ext_LT_ECL), und

- ECL-Berechnung für Buckets (get_ECL)

unterstützt.

🔍 Detaillierte Analyse

---

1. Funktionale Korrektheit

Stärken

✅ Mathematische Logik ist korrekt:

- Die Berechnung des 12M-ECL (get_12M_ECL) folgt der Formel:

```

ECL = Exposure LGD PD / (1 + EIR)^jahre_echt

```

- Die langfristige ECL-Berechnung (get_LT_ECL, ext_LT_ECL) berücksichtigt marginale PDs über die Laufzeit.

- Die Amortisierungslogik (get_ead_f) unterstützt Annuität und lineare Amortisierung korrekt.

✅ Interpolation für jährliche Daten wird unterstützt:

- readcsv2 ermöglicht lineare und quadratische Interpolation für monatliche Daten aus jährlichen Inputs.

Schwächen & Risiken

❌ Fehlende Eingabevalidierung:

- Es wird nicht geprüft, ob PD_cube existiert oder ob die Eingabeparameter (z.B. Month_Remaining, rating) gültig sind.

- Beispiel:

```r

PD_1 <- PD_cube[rating,rem_time,port] # Keine Prüfung, ob PD_cube existiert!

```

❌ Globale Abhängigkeiten:

- Die Funktionen hängen von globalen Variablen wie PD_cube, interpol, pdtime ab. Dies erschwert die Wiederverwendbarkeit und Testbarkeit.

❌ Unklare Logik in ext_LT_ECL:

- Die Funktion ext_LT_ECL nutzt row_i als Eingabe, aber die Abhängigkeiten zu ead_col und lgd_col sind unklar.

- Empfehlung: Die Funktion sollte explizit dokumentiert werden, welche Spalten im Datenrahmen row_i erwartet werden.

📚 Beispiel: Überarbeitete Version von get_12M_ECL

#' Berechnet den 12-Monats Expected Credit Loss (ECL)
#'
#' @param Month_Remaining Integer, verbleibende Monate bis zur Fälligkeit.
#' @param rating Character, Rating des Kreditnehmers.
#' @param exposure Numeric, Exposure-At-Default (EAD).
#' @param LGD Numeric, Loss Given Default (in %).
#' @param EIR Numeric, Effective Interest Rate.
#' @param amort Character, Amortisierungsmethode ("Annuity", "Linear", "None").
#' @param port Integer, Portfolio-ID.
#' @param PD_cube Array, Probability of Default Cube (Rating x Monat x Portfolio).
#' @return Numeric, berechneter ECL.
#' @examples
#' get_12M_ECL(12, "A", 1000000, 0.45, 0.05, "Annuity", 1, PD_cube = load_pd_cube())
get_12M_ECL <- function(Month_Remaining, rating, exposure, LGD, EIR, amort, port, PD_cube) {
  # Eingabevalidierung
  stopifnot(
    Month_Remaining >= 0,
    LGD >= 0 && LGD <= 1,
    EIR >= 0,
    port > 0
  )

  if (Month_Remaining == 0) {
    return(0)
  }

  jahre_echt <- min(Month_Remaining / 12, 1)
  rem_time <- min(Month_Remaining, 12)

  # PD aus dem Cube laden
  PD_1 <- PD_cube[rating, rem_time, port]

  # EAD-Faktor berechnen
  exp_f <- get_ead_f(EIR, jahre_echt, 0, amort)
  ead <- exposure * exp_f

  # ECL berechnen
  ecl <- ead * LGD * PD_1 / (1 + EIR)^jahre_echt

  return(ecl)
}
