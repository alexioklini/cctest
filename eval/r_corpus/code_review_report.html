3. Allgemeine Empfehlungen für alle Skripte

3.1 Portabilität & Konfiguration

- Problem: Hardcodierte Pfade und globale Variablen machen die Skripte nicht portabel.

- Lösung:

- Pfade als Parameter oder Umgebungsvariablen übergeben (z. B. über options() oder .Renviron).

- Globale Variablen in einer Konfigurationsdatei (z. B. config.yml) auslagern.

- Beispiel für Parameterisierung:

```r

# In ECL_122025.R

config <- list(

pfad = "W:/COLLABORATION/Risk_Public/IFRS 9/Berechnungen_Wertminderung",

pdtime = "PD_input_monthly",

interpol = "First_order"

)

source(paste(config$pfad, "/R codes/ECL_Hilfsfunktion_2.R", sep=""))

```

3.2 Fehlerbehandlung & Validierung

- Problem: Keine Prüfung auf NA/NULL oder ungültige Inputs.

- Lösung:

- Input-Validierung mit Paketen wie assertive oder checkmate:

```r

library(checkmate)

assert_file_exists(pfad)

assert_matrix(M_cpd)

```

- Fehlerbehandlung mit tryCatch:

```r

tryCatch({

df <- read.csv2(pfad, ...)

}, error = function(e) {

stop("Fehler beim Laden der Datei: ", e$message)

})

```

3.4 Performance

- Problem: Schleifen und mapply können langsam sein.

- Lösung:

- Vektorisierung mit dplyr oder data.table prüfen.

- Caching von Zwischenergebnissen (z. B. PD_cube).

- Parallelisierung mit parallel oder future-Paket.

3.5 Abhängigkeiten

- Problem: Fehlende oder undokumentierte Abhängigkeiten (z. B. bayes-Funktion).

- Lösung:

- Alle Abhängigkeiten explizit auflisten (z. B. in DESCRIPTION-Datei für Pakete).

- Fehlende Funktionen implementieren oder als externe Skripte bereitstellen.

3.6 Plots & Exporte

- Problem: Aktive Plots und redundante Excel-Exporte.

- Lösung:

- Plots optional machen:

```r

if (interactive()) {

plot(...) # Nur bei interaktiver Session

}

```

- Unnötige Exporte entfernen oder in separate Skripte auslagern.

3.7 Refactoring-Vorschläge

1. Gemeinsame Module erstellen:

- ecl_helpers.R für ECL_Hilfsfunktion.* und ECL_122025.R.

- pd_helpers.R für PD-Berechnungen (PIT_122025.R, TTC_122025.R).

- tl_helpers.R für Stage-Allokation (TL_122025.R).

2. Funktionen standardisieren:

- Gleiche Parameter-Namen und -Reihenfolge (z. B. exposure, LGD, EIR).

- Globale Variablen vermeiden (z. B. PD_cube als Parameter übergeben).

3. Tests hinzufügen:

- Unit-Tests mit testthat für kritische Funktionen (z. B. get_12M_ECL).

- Beispiel:

```r

test_that("get_12M_ECL returns correct ECL", {

expect_equal(get_12M_ECL(12, 1, 1000, 0.5, 0.05, "Linear", 1), 47.619)

})

```
