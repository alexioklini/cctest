## What got fixed + why (the real causes)

1. **Audio overview assumed 2025** (`451edeb`) — script-gen prompt never told the model the date.
   Injected `TODAY'S DATE` into `engine/audio_overview.py:_build_script_prompt`.

2. **ChromaDB HNSW corruption** (`Error finding id`, hung `tool_mempalace_query`) → switched dev
   box to **sqlite_exact** (exact cosine, no index → corruption structurally impossible). At dev
   scale (~17k) warm query ≈80ms; it's brute-force/linear so NOT for prod (→ Qdrant plan).

3. **Policy KG = 0 triples** — THE big one, root-caused after **3 wrong theories** (dead provider
   / retired model / evals-don't-use-KG — all disproven by reproduction). Real cause:
   **GDPR `anonymise` policy**; the `date` PII detector (category `personal`) fired **215×** on the
   date-saturated German bank-policy PDFs → anonymiser gutted each chunk (`tokens=1`) → extraction
   model returned empty → logged `"sidecar returned no reply"`. Two amplifiers fixed (`5a728be`):
   failed extraction no longer advances the progress cursor (retries instead of permanent-zero);
   loud "EXTRACTION MODEL APPEARS BROKEN" daemon log. **Resolution: GDPR + classification disabled**
   per user. See `[[project_kg_gdpr_anonymise_incident]]` memory.

4. **Dead `mistral-experimental` provider refs** repointed to `mistral-direct` (brain.py TTS/
   transcribe seeds + voxtral audio-backfill gate at ~10576; doc_convert.py OCR default).

5. **Config Doctor** (`engine/doctor.py`) — detects exactly the misconfig class behind this
   incident: model/config-ref → non-existent provider / disabled model, provider gaps, MemPalace
   health, failing KG wings. Two Doctor bugs found+fixed during the session (false "disabled" on
   scoped model ids; reading `server_config['models']` which is empty — the live models dict is
   `brain._models_config`).

## HARD GOTCHAS (will bite if forgotten)

- **NEVER SIGKILL brain-agent** (corrupts SQLite/HNSW). Use `launchctl kill SIGTERM …` (KeepAlive
  auto-restarts) or `bootout`+`bootstrap`. Enforced by a PreToolUse hook.
- **`_server_config()` does NOT contain `models`** — the live models dict is `brain._models_config`;
  providers are in `server_config['providers']`. (This caused a Doctor false-positive.)
- **Embedding device must be `mlx` or `cpu`, NEVER `auto`/`coreml`** — CoreML = 100% NaN embeddings
  on this Mac (silent corruption).
- **`MempalaceConfig` reads `~/.mempalace/config.json`** (palace_path `/palace`), NOT the repo
  `config.json` (palace `/brain-sqlite`). Backend/embedding for the brain palace are driven by
  **env in the launchd plist**, not the config `backend` key (which only applies when paths match).
- **Each backend needs a FRESH palace dir** — `get_collection` raises `BackendMismatchError` if a
  dir holds another backend's artifacts (chroma.sqlite3 / sqlite_exact.sqlite3 / qdrant_backend.json).
- **Probe the RUNNING server, not standalone `python3 -c "import brain"`** — standalone has no
  server_config (returns defaults) AND fails to import (no `jwt` in mempalace venv). Use the
  system `/opt/homebrew/bin/python3` for brain imports; mint a JWT with `user_id` = the real user
  id (`17368b7961d3` for admin), claim field is `user_id` not `sub`.
- **REPRODUCE before asserting root cause.** I asserted the KG cause wrong 3× from log-reading;
  the user was right each time. The temp `[kg-diag]` probe + GDPR log lines (`tokens=1`) were what
  finally proved it.
- **config.json is gitignored** (per-machine) — config edits aren't committed; code is.
- **brain.py: compile-check after edits** (`python3 -m py_compile brain.py`) + confirm
  `/v1/status` version after restart.
- **JS edits: run `cd web/js && ./js_gate.sh`** (eslint + net-globals count + smoke). Adding
  globals = bump `.globals-count.baseline` in the same commit (did this: 1221→1224 for Doctor).
