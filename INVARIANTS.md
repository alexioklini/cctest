## GDPR / PII Pre-Submit Scanner

71 regex detectors, client + server, zero external APIs.

**NER layer (v9.4.0)**: spaCy German `de_core_news_md` (~120MB) eager at startup adds 3 rule_ids: `name`/`address`/`organisation` (all `contact` category, default `ignore`). Runs in `_pii_scan_text` AFTER regex + bare-id (regex wins on overlap). Shape gate `_passes_shape_gate` drops FPs: lowercase entities, digits-only, `_ORG_ACRONYM_BLOCKLIST` (DSGVO/IBAN/BGB). No `ner_enabled` switch ‚Äî category action controls surfacing. Runtime: Settings ‚Üí GDPR per-language pill via `GET/POST /v1/gdpr/ner-models` (admin, audited). Browser scanner stays regex-only. **German + English (v9.347.0, M9.3)**: `en_core_web_md` loads alongside `de` and `scan_text` unions both models' findings (de first ‚Äî feeds the proximity gates; en adds non-overlapping spans). English spaCy uses OntoNotes labels (PERSON/GPE), German WikiNER (PER/LOC), both in `_LABEL_MAP`; en absence is non-fatal (de-only fallback). **FP-hardening (v9.349.0)** ‚Äî measured in a live DD run where the web-consent dialog was flooded: English (any lang ‚àâ `_MAIN_MODEL_LANGS`) is a PERSON-only recall net ‚Äî it contributes ONLY person names (from both its PERSON and ORG labels ‚Äî en mislabels real names as ORG on adverse-media prose), each coerced to `name` and forced through the STRICT span gate `_strict_name_span_ok` (all tokens capitalised, ‚â•2 substantial, no stop/connective word); its GPE/LOC are dropped (Bulgaria/Balkan noise). German keeps its trusted loose path. `_NAME_NOUN_SUFFIX` was extended with German title-case-prose suffixes (ungen/tionen/igt/isch/‚Ä¶) ‚Äî verified against 48 real surnames, zero collateral. **Language detection at the seam (v9.350.0)**: `_dominant_lang(text)` (deterministic stopword counting) picks the dominant language's model as the TRUSTED main model; the other runs as the strict PERSON-only recall net. A trusted-model finding whose ¬±90-char window is confidently the OTHER language is dropped (`_window_lang`). So an English document is parsed natively (splits 'CEO John Smith of Acme Corp' correctly) instead of German-model garbage. Stopword lists are deliberately EXCLUSIVE (cross-lingual 'in'/'an'/'am' removed ‚Äî else a short German sentence mis-detects as English). Nobiliary particles (van/le/du) kill a span only OUT of particle position; country codes (USA/UK/EU) unconditionally. **Documented residual**: a majority-GERMAN report with embedded English terms stays German-parsed ‚Äî its title-case-prose FPs ('Erste Instanz', 'Sechsstellige Darlehen') are declined German adjectives whose suffix filtering would eat 11 real surnames (Richter/Winter/Walter/‚Ä¶), so they are NOT safely heuristic-solvable ‚Äî they need a POS-aware model (`de_dep_news_trf`). Typed prompts + normal prose are clean. FP upgrade: `de_core_news_lg`/`de_dep_news_trf`.

**Two mirrored regex impls**: `PIIScanner` (`web/js/utils.js`); `_pii_rules()`+`_pii_scan_text()`+`_pii_scan_bare_identifiers()`+`PII_RULE_CATEGORIES`+`PII_DEFAULT_CATEGORY_ACTIONS` (`engine/pii_ner.py`, re-exported). Regex scanner pure (lazy-imports brain only for config-action resolver). **Rule order in `_pii_rules` is a correctness invariant ‚Äî never reorder.** Py‚ÜîJS parity auto-checked by `tools/check_pii_js_parity.py` (category/action maps + rule_ids; `refactor_gate.sh` Gate 4b); regex bodies differ by dialect, not checked.

**Three tiers** (first-match-wins, overlap suppression): cloud secrets/keys ‚Üí national IDs w/ checksums (~30 countries) ‚Üí context-fallback + bare-id heuristic. **Rule-order invariants**: context-gated (DE Steuer-ID, NL BSN, HU TAJ) before generic bare-digit; `credit_card` AFTER all national-ID checksums (RO CNP/KR RRN are 13-digit Luhn-passing); `phone` AFTER national IDs (XXX-XXX-XXXX SIN/NHS would steal slot); `credit_card` has `(?<![+\d])` so `+CC` phone prefixes don't match. **Overlap suppression**: successful matches claim spans, failed validations DON'T (weaker rules re-scan inside invalid IBAN ‚Äî why Aadhaar/PESEL/Steuer-ID are context-gated).

**Routing**: hard-block raises pre-LLM **only when model non-local** (local bypasses, data on-prem). `gdpr_pick_model_for_background(model, texts, purpose)` = **single decision point** for non-interactive: scan ‚Üí audit `pii_detected` ‚Üí swap to fallback if configured (`pii_auto_fallback`) ‚Üí else `server_block` raises `GDPRBlockedError` (`pii_blocked`) ‚Üí else warn. Client `piiBlockActive(chat)` filters dropdown to local-only when `server_block` + scanner on + PII present; auto-swaps via `piiEnsureLocalModel()`. `is_model_local()` ‚Üí `_is_local_base_url()` matches localhost/127/0.0/RFC1918.

**Config** (`gdpr_scanner`): master toggle, `server_log`, `server_block`, `default_local_fallback_model`, 8 per-category actions, `rule_overrides`, `email_allowlist`. `block`‚Üí`warn` when `server_block` off. **Not detected**: personal names, addresses, ICD codes, generic passport/license without context.

**Web-Egress-Gate (v9.334.0, L4 Phase 1)**: `brain._gdpr_guard_web_args(tool_name, args)`, called at the ONE live dispatch choke point (`engine/llm_loop.py:dispatch_tool`, before TOOL_DISPATCH lookup) ‚Äî covers interactive, background AND scheduler turns. Active only when `get_request_context()._gdpr_mapping_id` is set (anonymising sessions); everything else untouched. Checks ALL string args recursively (web_fetch: also the URL) against the session's **known protected values** ‚Äî `mapping.forward` (originals), `mapping.reverse` (fakes/tokens), `pii_decisions` ledger (FP values exempt) ‚Äî with lowercase + URL-slug normalisation (space‚Üí`-`/`_`/`+`/`%20`, first/last-token pair for multi-word names). **NOT findings-driven**: with `contact=ignore` the name is neither in the mapping nor actionable ‚Äî the gate asks "what IS PII" via a fresh scan with gate-own category policy (`_WEB_GATE_PASS_CATEGORIES = {business_id, network}` pass so technical queries never trip; rule_overrides dropped; user-declared FP values suppress fresh findings). **Invariants**: (1) **PERSON** fakes ‚Üí ALWAYS refuse, in every mode (fake search = semantically empty or poisons evidence with strangers' data) ‚Äî *narrowed in v9.344.0: fakes whose rule_id's CATEGORY is in `_WEB_GATE_PASS_CATEGORIES` are auto-released instead, see M5 below; a mixed query still refuses on the person*; (2) error `web_query_blocked_pii` carries `value_kind`, NEVER the value (it goes back to the model); (3) fail-CLOSED on gate crash while a mapping is active; (4) web tools must NEVER appear in any future args-deanonymisation whitelist (L3) ‚Äî that would be silent egress; (5) **a REFUSAL path never returns translated (=real) args** ‚Äî translation happens only on paths that actually pass. Modes `gdpr_scanner.web_egress`: `refuse` (default) | `ask` (per-value consent, Phase 2 below) | `block_group` (chat worker additionally excludes `WEB_SEARCH_TOOLS` via exclude_tools; dispatch gate stays as defense in depth) | `allow` (originals pass audited, fakes still refused). Audit: `pii_web_blocked`/`pii_web_egress` (kinds+mode only). Clamp: `_GDPR_ANON_CLAMP` instructs "nicht pr√ºfbar (Datenschutz)", never "no results" for a refused search. Tests: `tests/test_web_egress_gate.py`.

**Web-Egress-Consent (v9.338.0, L4 Phase 2 ‚Äî the `ask` mode)**: brings web corroboration (evidence class E4) back behind ONE user consent. `_gdpr_guard_web_args` now returns `(refusal, args)` ‚Äî the returned args are the DISPATCH copy (wire/history keep the model's fakes; callsite `dispatch_tool`). Flow: (a) first blocked call opens ONE `AskUserQuestion`-style batch **per VALUE, not per query** (`brain._web_consent_ask`; ask_user blocking machinery `_ask_user_register`+Event, answered via `POST /v1/chat/answer`, rendered by the existing chat question card; the slot is registered BEFORE the emit ‚Äî race-free). Non-interactive turns (no `event_callback` / `current_bg_task`) can't ask ‚Üí refuse `reason=no_consent`. One unanswered dialog per turn (`_web_consent_asked` set on the RequestContext); timeout/cancel persists NOTHING. (b) answers ‚Üí ledger rows `turn_action=release_web/deny_web` with a **value-only NAMESPACED hash** (`_web_release_hash` = sha256(`web|`+norm) ‚Äî NOT sha256(rule_id|value); a consent row must never shadow the value's `anonymise` row in `get_session_pii_decisions`, the wire rewrite depends on that row's `fake_value`). Reader: `ChatDB.get_session_web_releases` (latest-per-hash; revoke = newer deny_web row). (c) a FAKE of a released original in the args is translated to the original **for the outgoing request only** (`_web_release_translate_args`, case-insensitive + aligned URL-slug forms); (d) results re-anonymise through the existing L3b seam ‚Äî the model NEVER sees the original. (e) released-value matching is variant-INTERSECTION (releasing "Bonnie M Stark" covers the registered L2 variant "Bonnie Stark" via the first+last slug; deny wins on overlap of different consents); denied values refuse with a "Freigabe verweigert" hint and never re-open a dialog; **unreleased PERSON fakes are still refused in every mode ‚Äî never silently translated** (v9.344.0: pass-category fakes ARE translated, by standing policy consent ‚Äî see M5). (f) consent rows surface in the GDPR history modal as statuses `web_released`/`web_denied` (pii-decisions-view; `panels_gdpr.js` per-row Freigeben/Widerrufen toggle, saved as release_web/deny_web via the existing `POST /v1/gdpr/decisions` ‚Äî no new endpoint). (g) audit `match=released` (pii_web_egress) / `match=denied` (pii_web_blocked).

**Organisation entity layer (v9.344.0, M4 / Welle-2-G2)**: companies get the entity treatment persons got in L2 ‚Äî ONE fake per company; every surface form (long/short form, the ALLCAPS form registry + sanctions lists use, URL slug, legal-form variants) renders the form-matching variant of that ONE fake. Normalisation/rendering: `engine/identity.py` (`org_tokens`/`org_structure`/`org_attach`/`org_render_variant`/`org_variant_pairs`/`org_acronym`); Mapping wiring: `pseudonymizer.py` (`mapping.entities[*].kind` ‚àà {`person`,`org`}; **legacy rows without `kind` deserialise as `person`**; `_entity_fake_organisation` registered in `_ENTITY_GENERATORS`; `organisation` added to `_seed_entities_in_text_order`'s rule gate and to `apply_known_values`' default categories). **Invariants**: (1) **the legal form is NEVER part of the entity key** (`Wiener Privatbank SE` ‚â° `Wiener Privatbank` ‚Äî the NER span drops it anyway); (2) **`org_attach` is strict** ‚Äî token equality, NO substring/prefix merge, NO fuzzy: a false merge of two real companies poisons a regulatory report (three real "Atlantic Trading" must not collapse onto one fake); (3) **the fake MIRRORS the group structure** ‚Äî a stem-prefix relation (`wiener privatbank` ‚äÇ `wiener privatbank immobilien`) means the subsidiary INHERITS the parent's fake stem, in both discovery orders; without it the parent‚Üîsubsidiary relation (which lives in the name containment) is erased; (4) **name-bearing words are NOT legal forms** ‚Äî stripping `Holding`/`Partner`/`Invest` collapsed the three 3SI sisters into one company; (5) **an empty stem means "not a company" ‚Üí the value is NOT faked at all** (skip in `pseudonymize_text`, never fall through to the string faker): this covers bare generic corporate words the NER mistags (`Trust`, `Schwestern`) and **public bodies / registries / check-lists** (`OFAC-SDN-Liste`, `Firmenbuch`, `Companies House`, `BaFin`) ‚Äî they are the checking INSTRUMENT, not the subject, and faking them costs the model the name of the list it matches against (a QUALITY regression, only visible once the `organisation` rule is active); (6) **no fake-pool token may itself be a generic corporate word** ‚Äî it would be re-scanned as fresh org PII and faked again (fakes-of-fakes breaks the reply deanonymiser). **Known limit (documented, not silent)**: `WPB` is not derivable from the stem tokens (intra-word split of the compound `Privatbank`); a more aggressive acronym rule would fake `HTML`/`USA`/`LEI`/`EBIT`. Tests: `tests/test_pseudonymizer_org_entities.py`.

**Auto-release for pass-category values (v9.344.0, M5 / Welle-2-G3)**: a fake whose rule_id's CATEGORY is in `_WEB_GATE_PASS_CATEGORIES` (`business_id` ‚äÉ `organisation`, `network`) is **translated fake‚Üíoriginal for the OUTGOING request instead of refused ‚Äî in EVERY mode, `ask` included** (the policy already consents to the category; asking per company name would be a question without decision content). The model never sees the original (translation lives only in the dispatch copy of the args); results re-anonymise through the L3b seam. **Why it must exist**: without it, M4 made company research impossible ‚Äî the gate passed `organisation` on a fresh scan, but once the name was faked the model only knew the fake, and fakes refuse (G3: each component correct, the composition fatal). **The type bridge**: `known_fakes` carries a **rule_id**, `_WEB_GATE_PASS_CATEGORIES` holds **categories** ‚Üí cross via `PII_RULE_CATEGORIES`. **Invariants**: (1) **PERSON fakes still refuse in EVERY mode**, `allow` included ‚Äî mutation-tested; (2) a query mixing a person fake with an org fake **refuses on the person**; (3) **known fakes are length-preservingly MASKED out of the gate's fresh-scan input** ‚Äî a shape-preserving fake gets re-detected as fresh PII (measured: the org fake "Marbach Textil" came back as `rule=name`), which would re-open the consent dialog and undo the release; ¬ß1 has already decided about them. Audit kind `policy_released` ‚Üí `pii_web_egress` (NOT `pii_web_blocked` ‚Äî it is an egress event, not a block), degradation tally `web_policy_released`. Tests: `tests/test_web_auto_release.py`.

**Ad-hoc egress protection without a project (v9.344.0, M10b / Welle-2-G13)**: the gate used to be **inert without an active mapping** ‚Äî but the MAJORITY of real KYC/DD/compliance chats ran project-less (no preset ‚Üí no turn-1 auto-anonymise ‚Üí no mapping), so the clear name reached the search engine in turn 1 with the gate switched off entirely. The gate's **fresh scan needs no mapping**, so it now runs whenever `gdpr_scanner.enabled`, project or not, and refuses/asks on person PII. **Invariants**: (1) auto-ANONYMISATION is unchanged ‚Äî still modal/sticky driven; M10b changes only WHEN THE EGRESS IS GATED, never when values are pseudonymised; (2) **scanner off ‚áí gate inactive**, exactly as before (the off switch stays the off switch); (3) **honest dependency**: a bare NAME is NER-only (there is no name regex) ‚Äî without the loaded spaCy model M10b does not protect names; the server loads them at boot. Held as a test, not hidden.

**Dispatch symmetry (v9.336.0, L3)**: the model thinks in fakes, the tools work on raw data ‚Äî neither knows about the other.
- **L3a args-deanonymisation**: `brain._gdpr_deanon_tool_args(tool_name, args)` at the dispatch choke point (`engine/llm_loop.py:dispatch_tool`), **AFTER** the web gate (order is an invariant ‚Äî the gate judges the MODEL's args, never back-translated ones). Recursively translates fakes/tokens ‚Üí originals, only for `GDPR_ARGS_DEANON_TOOLS` (mempalace_query, mempalace_kg_*, read_document, read_file, list_directory, search_files, execute_command, python_exec, ocr_*, xlsx_*, text_diff, doc_checks tools). **Web tools are NEVER whitelisted** (silent egress) ‚Äî guarded by `tests/test_dispatch_symmetry.py` (whitelist ‚à© `WEB_SEARCH_TOOLS` = ‚àÖ). Returns a NEW structure; wire/history keep the fakes. Fail direction: tool runs with fakes (may miss, can never leak). Hardening beyond the handover: `execute_command`/`python_exec` strings containing network markers (`_DEANON_NETWORK_MARKER_RE`: curl/wget/https:///urllib/requests/‚Ä¶) keep their fakes ‚Äî a shell/script can reach the net itself, deanonymising there would be egress through a side door.
- **L3b results-anonymisation** (per-tool, NEVER a generic post-hook ‚Äî double-anon risk): `tool_mempalace_query` + `tool_mempalace_kg_query/search/neighbors` (v9.96.0 "stay raw" decision REVERSED) and web-inbound (`tool_web_fetch` via `_web_result_anon` wrapper ‚Äî content field only, covers cache/academic/YouTube/audio/file paths; `_searxng_query` both content returns ‚Üí all 5 searxng tools; `exa_search`) now pass `_gdpr_anon_tool_text`. Side effects (intended): the classification gate now also covers mempalace drawers + web content (daemon callers are safe ‚Äî no session model ‚Üí gate no-ops); the Websuche basket prefetch (`_build_web_sources`) inherits the seam (F5 partly closed) and catches `GDPRBlockedError` per source. Pseudonymised `read_path`s round-trip via L3a on the follow-up `read_document`.
- **L3c ledger-rewrite notice split**: `_apply_pii_decisions_to_wire` applies `_split_attachment_notice` like the scan path ‚Äî the ledger replace must never rewrite values inside attachment-notice disk paths (breaks follow-up reads).

**Entity-consistent pseudonymisation (v9.337.0, L2)**: the mapping works on ENTITY level, not string level ‚Äî ONE fake identity per person, every surface form maps to the FORM-MATCHING variant of the same fake.
- **Bookkeeping vs working memory**: `Mapping.entities` (`pseudonymizer.py`; {sur, givens, fake_sur, fake_givens}; encrypted-persisted, legacy rows load as `{}`) is the bookkeeping; the `forward`/`reverse` string tables stay the working memory. **Predictable variant PAIRS are registered as REAL forward/reverse entries** (`_register_entity_variants` ‚Üê `identity.standard_variant_pairs`: order/comma/initials/ALLCAPS/MRZ-name-form/glued-givens/surname) ‚Äî that is what makes the L3a args-deanon and the web-egress gate entity-aware with ZERO code change there (handover ¬ß7.9). Matching/rendering logic lives in `engine/identity.py` (shared with L1 doc_checks ‚Äî extend there, don't duplicate): `entity_attach` (3 tiers: names_match ‚â•0.84 ‚Üí initial-tolerant ‚Üí garble rescue `GARBLE_FLOOR=0.60`/`GARBLE_ANCHOR=0.72`, every token must bind a DISTINCT entity token), `render_variant` (token-wise, separators/digits/titles verbatim, case + initial style preserved).
- **Seeding order invariant**: `_seed_entities_in_text_order` runs BEFORE the end-descending splice pass ‚Äî real documents put the clean scan first and OCR-garble duplicates last; splice order would seed the entity from the worst garble (measured on the reference JPG set). Garbage guard: implausible name forms (non-name tokens, >3 givens) never create entities and never teach the learn path ‚Äî they fall back to the plain `_fake_name` shape fake.
- **Passport/MRZ (L2b)**: `passport`/`passport_ctx_loose`/`mrz`/`dob` joined `SHAPE_PRESERVING`. Keyword spans keep the keyword, only the value is faked. Bare passport number + 10-char number-with-check form are registered as own entries ‚Üí VIZ and MRZ carry the SAME fake. New scanner rule `mrz` (line-start anchored, structural validator, checksum trust tier; deliberately NO `$` anchor ‚Äî real OCR lines carry trailing garble). `_fake_mrz` rebuilds the line consistently: fake number, DOB shifted by the session offset, **expiry UNCHANGED** (document-lifecycle), nationality/sex verbatim, ALL ICAO-9303 check digits (incl. composite) recomputed via the doc_checks calculator ‚Äî the LLM's own MRZ math verifies again (F2). An opaque token claimed by another rule (cz_rc matches bare 9-digit numbers) is NEVER spliced into an MRZ line.
- **Date offset (L2c)**: `_fake_date` applies a CONSTANT salt-derived per-mapping offset (`date_offset_days`, ¬±5..25, real calendar arithmetic) instead of per-value day jitter ‚Üí ordering, deltas ("10y ‚àí 1d"), renewal gaps, EXIF distances stay EXACT. Formats (L2d) include textual months EN/DE + EXIF (`_DATE_PATTERNS` AND the scanner `date`/`dob` rules ‚Äî else the same date lives in two truths). Unparseable ‚Üí opaque token (the old bare-year fallback would let `reverse['1947']` rewrite every year occurrence).
- **Known-values sweep**: `apply_known_values` (word-bounded, exact registered values ‚â•4 chars; categories default `name/email/passport/dob` since L5) runs in `_gdpr_anon_tool_text` after the scan pass AND (since L5) on the chat worker's typed half ‚Äî the German spaCy NER regularly misses English names in mempalace drawers/web results, and a bare seeded passport number/DOB has no context keyword for the regex rules. Word boundary keeps paths intact (`STARK_Bonnie‚Ä¶` untouched ‚Äî path integrity is L3a's job) and compounds safe (`Starkstrom`). Audit event carries `known_values_swept`.
- Tests: `tests/test_pseudonymizer_entities.py`. Lone given names are deliberately never registered as variants (too FP-prone); the lone SURNAME is registered in Title AND ALLCAPS case (VIZ surname line) ‚Äî word-bounded only.

- **L5a ‚Äî OCR block is content, not boilerplate**: the deterministic image-description block (`_OCR_BLOCK_MARKER = "\n\n[Bild-Anh√§nge ‚Äî automatisch, ohne KI erkannt "`, deliberately NOT in `_ATTACH_NOTICE_PREFIXES`) is built BEFORE the path notice, so `_split_attachment_notice` lands it in the scannable typed half; for pre-L5 history (block INSIDE the notice) the split moves it into the typed half (wire-only reorder). **Report fidelity (v9.340.0, L6)**: de-anonymisation you can SEE fail ‚Äî no silent fake data in deliverables (F6).
- **Reverse linter** `pseudonymizer.lint_residual_fakes(text, mapping)` runs on FINAL (post-reverse) text at BOTH reverse seams and reports residual fake substance: `token_remnant` (this mapping's salt, or saltless `<KIND_N>` for minted kinds only), `exact_fake` (word-bounded reverse keys ‚Äî impossible right after `deanonymize_text` (fixed-point replace) but real on FILES: fakes split across docx runs, skipped xlsx formulas; token-shaped keys are check 1's, never double-reported), `reformatted_date` (the fake date in another surface form ‚Äî numeric ¬± padding + textual months DE/EN; parsed via `_parse_date_surface`), `name_genitive`/`name_initials` (from `mapping.entities`; genitive only when surname+s is not itself a reverse key ‚Äî plain substring replace restores that case correctly). Warning layer: recall over precision, findings carry FAKES only (safe to display), cap 50. Never blocks the write/reply path.
- **Seams**: reply (chat worker after `deanonymize_text`) ‚Üí `unrestored` in the synthetic `deanonymise_text` result + `metadata.gdpr_unrestored` + a persistent user-visible "‚öÝÔ∏è Datenschutz-Hinweis" block appended to the reply (nudge-hint pattern); files (`make_gdpr_after_file_write_cb`) ‚Üí lint of the written file's EXTRACTED text after `deanonymize_file` (`_lint_written_file`: plain exts direct read, office via `extract_attachment_text`, 20MB cap, best-effort) ‚Üí `unrestored`+`residues` in the `deanonymise_file` row.

Source of truth: `TOOL_DEFINITIONS` (`engine/tool_schemas.py`, Anthropic flat shape, re-exported on brain). Impls in `engine/tools/*` + `engine/mempalace_glue.py`; wiring (`TOOL_GROUPS`, `TOOL_DISPATCH`) in `brain.py`. Groups: core, documents, code_graph, web, email, delegation, git, scheduler, mcp, skills, nodes, context, memory, code_exec. Per-turn resolution: `resolve_active_tools(purpose=...)` ‚Äî single decision point (chat, scheduler, warmup, background, settings UI).

**Dispatch path**: sidecar `tool_use` ‚Üí POSTs Brain `/v1/tools/call` (nonce-protected via `sidecar.tool_endpoint_internal`, localhost-only) ‚Üí `tool_mcp.handle_tools_call` validates nonce, rebuilds context (current_agent, mcp_manager, current_session_id, current_user_id, project), dispatches to `engine.TOOL_DISPATCH` (or MCP fallback), captures result via `sidecar_proxy.capture_tool_result(turn_id, tool_use_id, ...)` for the `tool_dispatch_done` SSE ‚Üí returns result string. **Synchronous by design** ‚Äî `handle_tools_call` returns before the proxy drains `tool_dispatch_done`; don't make async without rethinking the result-capture handoff.

### Per-tool settings (admin-editable, global)

`config.json ‚Üí tool_settings`, keyed by tool name, loaded into `engine._tool_settings` + mirrored to `server_config`. Per-record schema:
- `enabled` (default True) ‚Äî global kill switch; false hides from every agent unless overridden. Server-internal callers unaffected.
- `deferred` (default False) ‚Äî hide from initial list, expose via `tool_search` only.
- `purposes` (default []) ‚Äî allowed purposes (`interactive`/`transform`/`memory_summary`/`research_minimal`); empty = all. Seeded from current behavior.
- `description`/`when_to_use`/`warnings`/`examples` (prose, empty=omitted) ‚Äî injected into system prompt under `## <tool> / ### <Section>` by `_render_tool_descriptions`.
- `applies_with` (list) ‚Äî all-of gate; prose renders only when every listed tool is also active.

Adding a prose record never hides/defers the tool. Renderer skips `enabled=false` defensively.

**Resolution hierarchy** (per LLM call): `effective_enabled/deferred = global`; if agent_id, `tool_overrides.<name>` overrides enabled/deferred (if present); drop if `not enabled`; drop if `call.purpose not in global.purposes` (when set); if deferred and not in discovered_tools ‚Üí drop (surface via tool_search). **Purpose layer is global-only** ‚Äî agents can't override (purpose is a property of the call). Scheduled tasks use same hierarchy; purpose from `tool_profile` (`""`‚Üí`research_minimal`, `"interactive"`‚Üí`interactive`) or name prefix (`_memory_summary_*`‚Üí`memory_summary`).

**Endpoints** (admin-only): `GET /v1/tools/settings` (all 64 tools + group + purposes), `POST /v1/tools/settings` (one record, validates name/applies_with/purposes, audited `tool_settings_save`), `GET /v1/tools/breakdown?agent=<id>` (per-tool token cost: name/description/schema), `GET/POST /v1/research-mode/disciplines` (refusal/precision/citation strings for research-mode project chats; per-section opt-out via empty string; audited `research_mode_disciplines_save`).

Legacy `tools.md` is gone ‚Äî anchored blocks one-shot migrated into `tool_settings` on first post-migration startup (`migrate_tool_settings_from_md`).

**Project-flow text** (3-step retrieval, `read_path` how-to, KG decision rule, BINARY DOCUMENTS note) lives in per-tool descriptions of `mempalace_query`/`read_document`/`mempalace_kg_search`/`mempalace_kg_query` ‚Äî NOT in `_build_system_prompt`. KG + read_document descriptions carry `applies_with: ["mempalace_query"]` so they only render in project-retrieval. Brain emits only a short "project chat with own memory" paragraph.

**Admin UI** (`settings_tools.js`, Settings ‚Üí Tools): grouped registry of all 64 tools. Per-tool panel: enabled/deferred toggles, group label, integration knobs (~13 tools w/ `tool_config`), 4 prose textareas, applies_with multi-select. Two saves: `Save` ‚Üí `/v1/tools/settings`, `Save integration` ‚Üí `/v1/tools/config`.

**Topic A/B split** (v9.0.x): Topic A = retrieval discipline (search-first, query shape, saving) in `tool_settings.mempalace_query.description`, renders for every chat w/ the tool, admin-editable. Topic B = output-format discipline (refuse-on-empty, precision, per-claim citation) in `DEFAULT_PROJECT_INSTRUCTIONS` constant, renders only project + research_mode, hardcoded.

**Constraints**:
- `execute_command`: no TTY/stdin, `TERM=dumb`. Banned commands in its description.
- Memory is MemPalace **direct, not MCP**: `mempalace_query` (+ `save_chat_to_memory`, `mempalace_get_drawer`, `mempalace_list_drawers`).
- **Adding a tool** = 4 sites / 3 files: schema in `TOOL_DEFINITIONS` (`engine/tool_schemas.py`), `TOOL_GROUPS` (`brain.py`), the `tool_*` fn (`engine/tools/<group>.py`, reaches brain via lazy `import brain as _brain`), `TOOL_DISPATCH` entry (`brain.py`). **Dispatch-identity rule**: `TOOL_DISPATCH` value must be a direct fn ref, not a `lambda args: tool_X(args)` forwarder, or the 4-site checks fail. Prose added later via UI.
- **Pre-existing bug**: 4 tools (`memory_delete`/`memory_recall`/`memory_shared`/`memory_persist`) missing from `TOOL_GROUPS` ‚Üí surface as `(ungrouped)`.

### Project Mode: `research_mode` (v8.31.0, split v9.0.x)

`project.json.research_mode` (bool) gates output-format discipline (Topic B) independently of `instructions`. Per-session `sessions.research_mode_override` (sticky NULL/0/1) layers on top. Effective = `override if not None else project.research_mode`. Resolution in `_build_system_prompt` (sets `research_mode_override` so `handlers/chat.py` gates citation validator/re-round consistently).

**Topic A/B split**: Topic A (search-first, query discipline, 3-step retrieval flow, `read_path`/`.md`/BINARY DOCUMENTS, KG decision rule) lives in `tool_settings.{mempalace_query,read_document,mempalace_kg_search,mempalace_kg_query}.description`, gated by tool presence + `applies_with: ["mempalace_query"]` (project chats only), admin-editable. Topic B (REFUSAL/PRECISION/CITATION) lives in `config.json ‚Üí research_mode_disciplines` (admin-editable, `GET/POST /v1/research-mode/disciplines`, per-section opt-out via empty string, defaults `RESEARCH_MODE_DISCIPLINE_DEFAULTS`), gated by project + research_mode.

**Mode ON** (Q&A/policy/compliance): soft `PROJECT MEMORY` block ("MUST consult memory tools first"); detailed flow via tool descriptions; 3 discipline sections injected; server-side citation validator + synchronous re-round on violation (>30% uncited or ‚â•2 unverified quotes), gated by `mempalace.citation_reround.enabled`.

**Mode OFF** (codegen/drafting): soft `PROJECT MEMORY` ("use mempalace_query when relevant"); Topic A tool descriptions still render; `research_mode_disciplines` NOT injected (model falls back on training); validator + re-round skipped.

- **Owner `instructions`** is purely additive in both modes (appended verbatim). Never a fallback for disciplines (that was v8.23, replaced).
- **Legacy migration**: `_project_research_mode(cfg)` ‚Äî absent `research_mode` field: empty `instructions` ‚Üí True; non-empty ‚Üí False.
- **Composer button** (`btn-research-mode`, hidden non-project): two-state cycle project default ‚Üî override-opposite; sticky like `save_to_memory`.
- **Infrastructure facts** (3-step flow body, BINARY DOCUMENTS, `read_path` vs `read_path_original`) STAY in system prompt regardless of mode.
- `DEFAULT_PROJECT_INSTRUCTIONS` constant = Topic-B text, injected only when research_mode on.
- **Anti-room-guessing**: `mempalace_query` `room` param enumerates real rooms (`general`/`artifacts`/`chat`/`chat_summary`/`chat_attachment`/`reference`) and forbids invention.
- **Sampling for Mistral Small** (gitignored): `temperature: 0.2`, `top_p: 0.85`. `0` rejected by provider; `0.1` no improvement.

## Project Knowledge Graph

LLM-driven document ‚Üí triples over project input folders + attachments. Post-pass after drawer mining; writes to `<palace_path>/knowledge_graph.sqlite3`.
- `kg_extract.py` ‚Äî Profiles: `normative` (policies/regs/SOPs), `generic` (prose). Chunking: **`source_file`** (default) re-chunks original markdown at `source_chunk_chars` (3500) ‚Äî ~70√ó yield. `inference_max_tokens=8000` (reasoning models exhaust mid-JSON below).
- `doc_convert.py` ‚Äî pre-mine binary‚Üícompanion `.md` under `<folder>/.brain-extracted/<name>.<ext>.md`. Idempotent via `(mtime,size)` frontmatter. `<!-- brain-source: <abs> -->` resolves back to original.
- **`normative` vocabulary**: 12 controlled English predicates (requires, forbids, permits, defines, cites, applies_to, effective_from, supersedes, responsible_party, condition, exception, penalty). Subjects/objects verbatim in source language. Off-vocab allowed.
- **Daemon hook**: `_run_kg_for(...)` resolves prefix via `os.path.realpath()` (macOS `/tmp`‚Üí`/private/tmp`, else source_files don't match).
- **Closet regen** (optional): replaces regex closets w/ LLM topic lines. Incremental via `closet_regen_progress` cursor on `(mtime,size)`.
- **Source-change invalidation**: snapshots `kg_extraction_source_state`; on diff DELETEs `triples` matching **exact** `source_file` + progress rows + orphan-entity sweep.
- **Agent KG tools**: auto-scope to caller's project via `project`; refused outside project context.

**Known constraints**: (1) GPU: 26B warmpool (~22GB) + e4b extraction (~5GB) exceeds oMLX 25.6GB ‚Äî raise cap, or 26B `warmup:false` + `max_concurrent:1`, or cloud. (2) MemPalace KG 3.3.0 lacks `source_drawer_id`+`adapter_name` ‚Üí `kg_extract` falls back via `TypeError` to legacy 5-arg `add_triple`. (3) KG path is `<palace_path>/knowledge_graph.sqlite3`, NOT `~/.mempalace/...`. (4) Code skipped (`_is_code_path()` ‚Üí folded into Brain's code graph).

## MemPalace (Direct Integration)

Imported as Python package ‚Äî no MCP, no subprocess.

**Vocabulary**: **Drawer** = atomic verbatim chunk (~800 chars, content-hash id). **Closet** = index layer (topic|entities|‚Üídrawer_ids) boosting search. **Room** = topic bucket. **Wing** = namespace. **Hall/Tunnel** = graph edges (future).

**Wing scheme** (ID-only): `user__<uid>` (private), `team__<tid>` (team-shared), `project__<pid>` (strictly isolated), bare names (shared, anyone reads). `_resolve_session_wing` priority: project ‚Üí team ‚Üí user ‚Üí empty. Anonymous ‚Üí `""`, skipped by chat-sync.

`mempalace_query`: when `project` set, **force-scopes** to `project__<id>` (refuses if id missing rather than leak). Else defaults `user__<current_user_id>`. Unspecified-wing visibility filter: drops `project__*`, matches `user__/team__` against caller, treats untyped as shared.

**Chat sync classifier gate**: LLM classifies message pairs before filing. `fact`/`preference`/`decision`/`reference` filed; `generic`/`refusal`/`chitchat` skipped. Non-streaming, `max_tokens:20`, fail-open. Per-session 3-state: `0=off`, `1=on`, `2=auto`. `save_chat_to_memory` tool enables on "remember this". Per-turn control via palace-icon menu ‚Üí `memorize_turns`/`purge_turns` (`turn_ids` or `{scope, anchor_turn_id}`). Disable-with-purge prompt when toggling on/auto ‚Üí off w/ drawers present.

**Session delete cleanup**: `delete_session` purges drawers + closets `source_file LIKE session/<sid>%`. **Archive leaves drawers intact.**

**Daemon 1 ‚Äî `mempalace-miner`** (1800s default): autonomous artifact ingestion. Walks `AGENTS_DIR`, classifies by folder: `sched-` ‚Üí scheduled, else chat.
- Sched: file only output-role files via `tool_add_drawer`, skip intermediates. `source_file=session/sched-<run>#artifact/<name>`, wing `<agent_id>_artifacts`. Sched chat content (reasoning/tool calls) stays out.
- Chat: gated on parent `save_to_memory > 0`. Ensures `mempalace.yaml` (marker `# managed by brain-agent server.py`).
- Startup `_purge_orphan_chroma_queue()`: detects HNSW segments missing `max_seq_id`, deletes >24h `embeddings_queue` rows.
- plist `PYTHONUNBUFFERED=1` so `[mempalace-miner]` reaches log immediately.

**Daemon 2 ‚Äî `mempalace-chat-sync`** (60s default): mirrors to wings:
- Chat turns ‚Üí `room=chat`, `source_file=session/<sid>#turn/<user_msg_id>` (anchor = opening user msg DB id)
- Summaries ‚Üí `room=chat_summary`, content-hashed
- Attachment metadata (filename/mime/size, NOT bytes) ‚Üí `room=chat_attachment`
- Allowlisted tool_results (`exa_search`/`web_fetch`/`read_document`) ‚Üí `room=reference`
- Uses `mempalace.mcp_server.tool_add_drawer` (fn). Reads `MEMPALACE_PALACE_PATH` from `mempalace.palace_path` before import.
- **Closet rebuild** per dirty group: `purge_file_closets` + `build_closet_lines` + `upsert_closet_lines` (else chat memories miss closet boost). Gated by `mempalace.chat_sync.build_closets`.
- **Cursor**: `chat_mempalace_sync (session_id PK, last_message_id, last_summary_hash, updated_at)` in chats.db.
- **Not mined**: attachment bytes, artifact version history, tool_result outside allowlist.
