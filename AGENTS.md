# Repository Guidelines

Welcome to the project! This guide outlines the repository structure, development workflows, and best practices to help you contribute effectively.

## Project Structure & Module Organization

The project is organized as follows:
```
<project_root>/
├── src/          # Source code
├── tests/        # Test files
└── assets/       # Static assets
```
## Build, Test, and Development Commands

Here are the key commands for building, testing, and running the project locally:

- `npm install`: Install dependencies.
- `npm test`: Run tests.
- `npm start`: Start the development server.
## Coding Style & Naming Conventions

Follow these guidelines for consistent code style:

- Use 2 spaces for indentation.
- Name files and variables in `camelCase`.
- Use descriptive names for functions and variables.

## Testing Guidelines

Tests are located in the `tests/` directory. Use `npm test` to run the test suite. Ensure all new features are covered by tests.
## Commit & Pull Request Guidelines

- Use descriptive commit messages.
- Link pull requests to relevant issues.
- Include screenshots or examples for UI changes.
