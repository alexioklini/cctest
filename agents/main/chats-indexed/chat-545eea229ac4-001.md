# Remote Node Setup Guide

Remote nodes allow Brain Agent to execute tools (shell commands, file ops) on other machines via HTTP long-polling.

## Architecture

```
Brain Agent Server (host machine, port 8420)
  ← HTTP long-poll ← node.py (remote machine)
```

The node connects **outbound** to the server — no inbound ports needed on the remote machine. The node polls `/v1/nodes/poll`, receives commands, executes them locally, and POSTs results back.

## Prerequisites

- **Remote machine**: Python 3.9+ (stdlib only, no pip dependencies)
- **Network**: Remote machine must be able to reach the Brain Agent server IP:port
- **File**: `node.py` — single-file, self-contained node agent

## Step-by-Step Setup

### 1. Register the Node on the Server

Via API:
```bash
curl -X POST http://127.0.0.1:8420/v1/nodes -H "Content-Type: application/json" \
  -d '{"action": "add", "name": "MyNode", "description": "Mac Mini M4 24GB"}'
```

Response includes `token` (e.g., `nd_d8faf3fb550e529c167f3f75ec7e937a`) and an `install_command` template.

Or via the Web UI: Settings → Nodes → Add Node.

### 2. Copy `node.py` to the Remote Machine

```bash
scp node.py user@REMOTE_IP:~/node.py
```

### 3. Install as a launchd Daemon (macOS)

```bash
python3 ~/node.py --install \
  --server http://SERVER_IP:8420 \
  --token nd_YOUR_TOKEN \
  --name MyNode
```

This creates a launchd plist at `~/Library/LaunchAgents/com.brain-agent.node.mynode.plist` with `KeepAlive` and `RunAtLoad` — the node auto-starts on boot and restarts on crash.

### 4. Verify Connection

```bash
curl -s http://127.0.0.1:8420/v1/nodes | python3 -m json.tool
```

The node should show `"status": "connected"` with hostname, OS, and system metrics.

## Critical: Server URL

The `--server` URL must be the **server machine's IP as reachable from the remote machine**, not the remote machine's own IP or `localhost`.

To find the server's LAN IP (on the server machine):
```bash
ipconfig getifaddr en0    # macOS
hostname -I               # Linux
```

Example: if the server runs on `192.168.1.221:8420` and the remote node is on `192.168.4.65`, use `--server http://192.168.1.221:8420`.

## Node CLI Reference

| Command | Description |
|---------|-------------|
| `python3 node.py --install --server URL --token TOKEN --name NAME` | Install as launchd daemon |
| `python3 node.py --uninstall --name NAME` | Remove launchd daemon |
| `python3 node.py --status` | Show all installed node services |
| `python3 node.py --server URL --token TOKEN` | Run in foreground (for debugging) |
| `python3 node.py --generate-token` | Generate a token locally (not registered on server) |
| `python3 node.py --allowed-paths /home/user,/tmp` | Restrict file ops to specific paths |

## Server API Reference

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/nodes` | GET | List all nodes with status, metrics, config |
| `/v1/nodes` | POST | Manage nodes: `action` = `add`, `remove`, `pause`, `resume`, `update` |
| `/v1/nodes/execute` | POST | Submit a command to a specific node |
| `/v1/nodes/poll?token=X` | GET | Node polls for commands (30s long-poll) |
| `/v1/nodes/result` | POST | Node returns command results |

### POST /v1/nodes Actions

**Add** — registers node, generates token:
```json
{"action": "add", "name": "MyNode", "description": "...", "allowed_tools": ["execute_command", "read_file", "write_file", "list_directory"], "tags": [], "max_concurrent": 5, "command_timeout": 300}
```

**Remove** — deletes node config and disconnects:
```json
{"action": "remove", "name": "MyNode"}
```

**Pause/Resume** — temporarily disable a node (node stays connected but receives no commands):
```json
{"action": "pause", "name": "MyNode"}
{"action": "resume", "name": "MyNode"}
```

**Update** — modify config fields:
```json
{"action": "update", "name": "MyNode", "description": "new desc", "allowed_tools": [...], "max_concurrent": 10}
```

## Available Tools on Nodes

Nodes support these tools (configurable via `allowed_tools`):

| Tool | Description |
|------|-------------|
| `execute_command` | Run shell commands (timeout capped at 600s, output capped at 50KB) |
| `read_file` | Read file contents with line numbers (supports offset/limit) |
| `write_file` | Write file contents (creates parent dirs) |
| `list_directory` | List directory entries (supports glob patterns, recursive) |
| `edit_file` | Edit files (if enabled in allowed_tools) |
| `search_files` | Search file contents (if enabled in allowed_tools) |

## Logs

On the remote machine:
```
~/.brain-agent/node-{name}.log        # stdout (connection status, commands executed)
~/.brain-agent/node-{name}.error.log  # stderr (Python errors)
```

## System Metrics

The node reports these metrics on every heartbeat (each poll cycle):
- CPU percent (load average / core count)
- Memory used/total (GB)
- Disk free (GB)
- Uptime (seconds)
- Active/total command counts

## Connection Behavior

- **Reconnect with exponential backoff**: 1s → 2s → 4s → ... → 60s max
- **Long-poll interval**: 30s server-side, 2s sleep between polls on no-command
- **Authentication**: token validated on every poll; 401 = invalid token (stops node), 403 = paused (retries in 10s)
- **Concurrency**: commands execute in daemon threads; `max_concurrent` is advisory (enforced server-side)
- **KeepAlive**: launchd restarts the node process if it exits

## Python Version Compatibility

`node.py` uses `from __future__ import annotations` for compatibility with Python 3.9+. The remote machine does not need any pip packages — only the Python standard library.
