**assistant**: Hier ist eine Übersicht über die aktuellen Modelle von Mistral AI, basierend auf den neuesten Informationen aus 2026:

### 1. **Mistral Large 3**
- **Anwendungsgebiete**: Komplexe Analysen, multimodale Aufgaben (Text, Bild), multilinguale Anwendungen, Reasoning, Code-Generierung.
- **Geschwindigkeit**: Hohe Performance, aber langsamer als kleinere Modelle.
- **Unterstützte APIs**: Vollständige Integration mit Mistral API, OpenAI-kompatibel, JSON-Modus, Function Calling.
- **Besonderheiten**: 41B aktive Parameter, 675B Gesamtparameter, großes Kontextfenster (128.000 Token).

### 2. **Mistral Small 4**
- **Anwendungsgebiete**: Effiziente Chat-Anwendungen, Codierung, agentenbasierte Aufgaben, schnelle Antworten.
- **Geschwindigkeit**: Sehr schnell, ideal für Echtzeit-Anwendungen.
- **Unterstützte APIs**: Mistral API, JSON-Modus, Function Calling.
- **Besonderheiten**: Kompakt, aber leistungsstark, kostengünstig.

### 3. **Codestral**
- **Anwendungsgebiete**: Code-Generierung, Code-Verständnis, IDE-Integration, Fill-in-the-Middle (FIM).
- **Geschwindigkeit**: Optimiert für Entwickler-Workflows, schnelle Code-Vervollständigung.
- **Unterstützte APIs**: Mistral API, FIM-Unterstützung, JSON-Modus.
- **Besonderheiten**: Spezialisiert auf Programmieraufgaben, 32K Kontextfenster.

### 4. **Pixtral**
- **Anwendungsgebiete**: Multimodale Aufgaben (Text + Bild), Dokumentenanalyse, OCR, Diagramminterpretation.
- **Geschwindigkeit**: Moderat, abhängig von der Komplexität der Eingabe.
- **Unterstützte APIs**: Mistral API, JSON-Modus, Function Calling.
- **Besonderheiten**: Vision-Language-Modell, 128K Kontextfenster.

### 5. **Mistral 7B**
- **Anwendungsgebiete**: Lokale Entwicklung, Open-Source-Projekte, kostengünstige Implementierungen.
- **Geschwindigkeit**: Schnell, aber begrenzt durch Hardware.
- **Unterstützte APIs**: Selbstgehostet, Apache 2.0 Lizenz, kompatibel mit Ollama/vLLM.
- **Besonderheiten**: Open-Source, 8K Kontextfenster, ideal für lokale Bereitstellung.

### 6. **Ministral (3B, 8B, 14B)**
- **Anwendungsgebiete**: Edge-Geräte, Robotik, selbstgehostete Systeme.
- **Geschwindigkeit**: Sehr schnell, optimiert für Edge-Deployment.
- **Unterstützte APIs**: Selbstgehostet, kompatibel mit verschiedenen Cloud-Anbietern.
- **Besonderheiten**: Multimodal, multilingual, kompakt und effizient.

### 7. **Magistral**
- **Anwendungsgebiete**: Spezialisierte Reasoning-Aufgaben, transparente und multilinguale Anwendungen.
- **Geschwindigkeit**: Moderat, abhängig von der Aufgabe.
- **Unterstützte APIs**: Mistral API, JSON-Modus.
- **Besonderheiten**: Transparente Entscheidungsfindung, ideal für Unternehmensanwendungen.

### 8. **Voxtral**
- **Anwendungsgebiete**: Audio-zu-Text, Spracherkennung, Sprachverarbeitung.
- **Geschwindigkeit**: Schnell, optimiert für Echtzeit-Spracherkennung.
- **Unterstützte APIs**: Mistral API, Audio-Integration.
- **Besonderheiten**: State-of-the-Art-Spracherkennung, ideal für Sprachassistenten.

### 9. **Mistral Embed**
- **Anwendungsgebiete**: Semantische Suche, Inhaltsorganisation, Embeddings.
- **Geschwindigkeit**: Schnell, optimiert für Suchanfragen.
- **Unterstützte APIs**: Mistral API, Embedding-Unterstützung.
- **Besonderheiten**: State-of-the-Art-Embedding-Modell für semantische Suche.

### 10. **Mistral Moderation**
- **Anwendungsgebiete**: Inhaltsmoderation, Sicherheit, Compliance.
- **Geschwindigkeit**: Schnell, optimiert für Echtzeit-Moderation.
- **Unterstützte APIs**: Mistral API, Moderations-Tools.
- **Besonderheiten**: Anpassbare Moderation in mehreren Sprachen.

### API-Unterstützung
- **Mistral API**: Vollständige Unterstützung für alle Modelle, inklusive Function Calling, JSON-Modus und Tool-Integration.
- **OpenAI-Kompatibilität**: Einfache Migration von OpenAI zu Mistral.
- **Cloud-Anbieter**: Verfügbar auf AWS Bedrock, Google Cloud Model Garden, Azure AI Studio, IBM, Snowflake, NVIDIA, Outscale.

### Geschwindigkeit und Leistung
- **Schnellste Modelle**: Ministral 3 3B (279 Token/Sekunde), Devstral Small (255 Token/Sekunde).
- **Langsamste Modelle**: Mistral Large 3 (hohe Leistung, aber langsamer).
- **Latenz**: Ministral 3 3B bietet die niedrigste Latenz (0.41s Time-to-First-Token).
