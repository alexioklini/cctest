**assistant**: ### Summary of the Inferencer Distributed Compute User Guide

#### Overview
- **Inferencer Pro** is an MLX-native inference server for macOS that supports distributed computing across two Apple Silicon Macs.
- **Key Features:**
  - Relay vertical architecture: One GPU is active at a time, passing layers back and forth.
  - Supports pairs of machines (maximum 2 machines).
  - Asymmetric RAM is supported.
  - Greedy layer distribution: Automatic, no manual configuration needed.
  - Average speed: Each machine runs at its own max speed; the result is the average of both.

#### Requirements
- **Hardware:**
  - 2 Apple Silicon Macs (Mac Studio, MacBook Pro, Mac Mini — any combination).
  - Both on the same local network (Ethernet recommended, WiFi viable).
  - Thunderbolt 5 direct connection is supported but has known stability issues.
- **Software:**
  - Inferencer Pro v1.10.4+ (macOS App Store).
  - macOS 26.x recommended.
  - The same model must be downloaded/accessible on both machines.

#### Architecture
- **Clients (unlimited):** n8n, Cline, VS Code, OpenWebUI, Claude Code (future).
- **Compute (pair):** Machine A (Server) and Machine B (Compute Node).
- **Roles:**
  - Machine B connects to Machine A, not the other way around.
  - Machine A orchestrates split inference and provides RAM + compute.

#### Setup Steps
1. **Configure Machine B (Compute Node):**
   - Open Inferencer Pro on Machine B.
   - Go to Server > Serve and start the server.
   - Note the hostname and port (e.g., ultra-256a.local:8081).

2. **Configure Machine A (Server):**
   - Open Inferencer Pro on Machine A.
   - Go to Server > Serve and start the server.
   - Configure settings in Inference Settings > Generation.

3. **Connect B to A:**
   - On Machine B, go to Server > Connect.
   - Machine A should appear via auto-discovery.
   - If not, enter the IP/hostname manually.
   - Click Connect and enable Auto-connect.

4. **Verify Connection:**
   - On Machine B's Connect panel, verify Machine A's details.
   - Machine B's UI should show Machine A's model list.

#### Loading a Distributed Model
- **Prerequisites:**
  - The model must be present on both machines.
  - Download the same model on both machines via Models > Hugging Face or copy the model files to both machines' local folders.

- **Loading Procedure:**
  - On Machine B, select the model.
  - Toggle Distributed Compute ON in the chat bar.
  - The model loads and distributes across both machines.

#### API Access
- **Endpoints:**
  - OpenAI-compatible: `http://[IP]:54321/v1/chat/completions`
  - Models list: `http://[IP]:54321/v1/models`
  - Ollama-compatible: `http://[IP]:54321/api/tags`
  - Anthropic Messages: `http://[IP]:54321/v1/messages` (planned)

- **Model Prefixes:**
  - `(none)`: Local model on Machine A only.
  - `<remote>`: Model on Machine B only.
  - `<distributed>`: Model split across A + B.

#### Performance Data
- **Distributed Benchmarks:**
  - **Speed Model:** Each machine runs at its maximum speed; the result is the average of both.
  - **Scaling:** Dense models scale better than MoE models.
  - **Comparison with LM Studio:** Inferencer Distributed is faster and more stable.

#### Troubleshooting
- **Server Not Accessible from Network:**
  - Check server is running.
  - Enable Inferencer in macOS Settings → Privacy & Security → Local Network.
  - Disable or allow Inferencer in the firewall.
  - Test locally: `curl http://localhost:54321/v1/models`.

- **Model Requires More Memory Than Combined Memory Available:**
  - Use a smaller quantization of the same model.
  - Enable cache quantization (Precision 4.5-bit).
  - Use a different, smaller model.

- **Unable to Setup Model for Distributed Compute:**
  - Verify both servers are running with Distributed Compute enabled.
  - Check Distributed Compute Port (default: 49152) is not blocked.
  - Restart both servers.

- **Model Loads on One Machine Only (No Split):**
  - Expected if the model fits on Machine A alone.
  - Verify the model is present on both machines.
  - Toggle Distributed Compute ON in the chat bar.

#### Known Limitations
- **2 Machines Max:** Pairs only. >2 machine support is on the roadmap.
- **1 Model at a Time:** No multi-model simultaneous inference.
- **Thunderbolt 5 Distributed Unstable:** Use Ethernet/IP as primary transport.
- **No Limit Context on macOS:** Available on Windows, coming to macOS.
- **MoE Scaling Limited:** 1.36x on 4 nodes vs 3.2x for dense models.
- **Model Must Be on Both Machines:** Distributed won't activate without the model on both nodes.

#### Roadmap
- **Prioritized Features:**
  - >2 machines over WiFi.
  - Pipeline batching.
  - Horizontal compute + RDMA.
- **Planned Features:**
  - Anthropic Messages API.
  - Multi-model simultaneous.
  - Limit Context macOS.
  - Dynamic Repetition Penalty.
  - Distributed >2 machines.
  - RDMA Thunderbolt.
