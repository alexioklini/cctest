## Logs

On the remote machine:
```
~/.brain-agent/node-{name}.log        # stdout (connection status, commands executed)
~/.brain-agent/node-{name}.error.log  # stderr (Python errors)
```

## System Metrics

The node reports these metrics on every heartbeat (each poll cycle):
- CPU percent (load average / core count)
- Memory used/total (GB)
- Disk free (GB)
- Uptime (seconds)
- Active/total command counts

## Connection Behavior

- **Reconnect with exponential backoff**: 1s → 2s → 4s → ... → 60s max
- **Long-poll interval**: 30s server-side, 2s sleep between polls on no-command
- **Authentication**: token validated on every poll; 401 = invalid token (stops node), 403 = paused (retries in 10s)
- **Concurrency**: commands execute in daemon threads; `max_concurrent` is advisory (enforced server-side)
- **KeepAlive**: launchd restarts the node process if it exits

## Python Version Compatibility

`node.py` uses `from __future__ import annotations` for compatibility with Python 3.9+. The remote machine does not need any pip packages — only the Python standard library.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `Connection refused` in node log | Wrong server URL or server not running | Verify `--server` URL points to server machine's IP, not remote's own IP |
| `Authentication failed: invalid token` | Token mismatch | Re-register node via API and use the new token |
| `Node is paused` | Node was paused via API/UI | Resume: `POST /v1/nodes {"action": "resume", "name": "..."}` |
| Node shows `disconnected` on server | Node process not running or network issue | Check `launchctl list | grep brain-agent` on remote; check logs |
| `TypeError: unsupported operand type(s) for \|` | Python < 3.10 without `__future__` annotations | Ensure `node.py` has `from __future__ import annotations` at top |
| Node connects then disconnects repeatedly | Firewall or timeout issues | Check network path; ensure port 8420 is open between machines |
```
