Prompt Caching\n\nInferencer supports prefix-matching prompt caching.

If consecutive requests share the same system prompt, only the first request processes it fully — subsequent requests hit the cache for a 99x speedup.\n\n## Performance Impact\n\nRequest 1:  ~90 sec  (full system prompt processing)\n\nRequest 2:  <1 sec   (cache hit — system prompt already processed)\n\nRequest 3+: <1 sec   each\n\nBatch of 33 requests (same system prompt):\n\nWithout cache: 33 × 90 sec = ~49 min\n\nWith cache:    90 + 32 × 1  = ~2 min\n\nSavings:       ~47 minutes per cycle\n\n## Configuration\n\n## What Invalidates the Cache\n\n# 8.

Cache Quantization (Precision)\n\nCache quantization reduces the memory consumed by conversation context by up to 3.55x.\n\nLocation: Inference Settings > Model > Precision\n\nAvailable levels: 16-bit (default), 9-bit, 8.5-bit, 6.5-bit, 5.5-bit, 4.5-bit, 3.5-bit.\n\n## Measured Data (~2,600 tokens)\n\nRule of thumb: For DeepSeek and Kimi models, always enable 4.5-bit cache quantization.

These models consume enormous amounts of RAM in long contexts without it.\n\n# 9.

Batching\n\nBatching enables 2-4+ simultaneous inferences.

Individual tokens/sec decrease, but total throughput increases.\n\nExample: 1 inference @27 t/s → 4 inferences @27.5 t/s each = 110 t/s total throughput.\n\n## Configuration\n\nVia UI: Toggle in Inference Settings > Generation.\n\nVia API (per-request):\n\n{\n\n\"model\": \"default\",\n\n\"messages\": [...],\n\n\"batching\": true\n\n}\n\n## Constraints\n\nIdentical temperature required for batched requests\n\nWorks with prompt caching (Batch Caching)\n\nFor sequential workflows (e.g., processing articles one by one): \"batching\": false is recommended\n\n# 10.

Context Window Management\n\n## Memory Impact by Model (~2,600 tokens)\n\nWarning: DeepSeek and Kimi consume massive RAM in long contexts without cache quantization.

For distributed pairs, this directly impacts how much context you can afford alongside the model weights.\n\n## Strategies (ranked)\n\nManual pruning — delete old messages (available now)\n\nDrop-the-middle — automatic when limit is reached (Windows only)\n\nCache quantization — Precision 16-bit → 3.5-bit (available now)\n\nDistributed compute — more RAM = more context (available now)\n\nLLM compaction — summarize old messages (not implemented)\n\n# 11.

Performance Data\n\n## Distributed Benchmarks\n\n## Speed Model\n\nEach machine runs at its maximum speed.
