## 🏗️ Architecture & Design

**Dense Transformer vs. Mixture-of-Experts (MoE):**

Devstral 2's most architecturally distinct choice is its **fully dense design**. While competitors like DeepSeek V3.2 (671B MoE) and Kimi K2 (1T MoE) route tokens through only a fraction of active parameters, Devstral 2 engages all 123B parameters for every token processed.

| Architecture Property | Devstral 2 (Dense) | DeepSeek V3.2 / Kimi K2 (MoE) |
|---|---|---|
| Parameters activated per token | All 123B | Subset (~active experts) |
| Inference predictability | High | Variable |
| Deployment complexity | Lower | Higher |
| Governance/auditability | Easier | More complex |
| Raw parameter count | 123B | 671B – 1T |

**Key design principles:**
- Optimized for **instruction following and code-specific information density**
- Trained to maintain **cross-file architectural coherence** (changes in one file reconcile with imports/usage sites elsewhere)
- Supports **file structure awareness, project dependency tracking, and version control state interpretation**
- Deployable on **NVIDIA DGX / NIM stacks** behind enterprise firewalls

8. **Hardware requirement** — flagship needs 4× H100-class GPUs; not accessible for all self-hosters
